DNA Motif Analysis & Prediction Pipeline

Developed with Gemini AI for Bioinformatics Research

This project provides a multi-stage pipeline for processing genomic data, training a classification model to distinguish between positive promoters and negative sequences (viral/junk), and predicting motifs in unknown DNA sequences.

🛠️ Environment Setup

Before running the scripts, ensure you are using VS Code and have a Python virtual environment activated.

1. Create and Activate Virtual Environment

2. Install Dependencies

Run the following command to install the necessary bioinformatics and machine learning libraries:

📂 Project Structure & Workflow

The pipeline must be executed in numerical order. Ensure your directory contains the following folders: positive_library, negative_library, negative, DNA_file_holder, and analysis_results.

Phase 1: Data Preparation

00_neg_junk_prep.py

  • Purpose: Generates a “noise” or junk DNA library.
  • Path Requirement: Uses positive_library\hg38_e4bBD_01.fasta as a reference to generate matching negative control data.
  • Output: negative_library\true_junk.fasta

00_neg_viral_prep.py

  • Purpose: Processes raw viral sequences.
  • Path Requirement: Place your 3 HERVK FASTA files into the negative/ folder.
  • Output: negative_library\viral_HERVK_01.fasta

Phase 2: Model Training

01_ai_train.py

  • Purpose: Trains the AI model using the libraries prepared in Phase 1.
  • Input Files: * positive_library/hg38_e4bBD_01.fasta
    • negative_library/viral_HERVK_01.fasta
    • negative_library/true_junk.fasta

Phase 3: Data Acquisition (Optional)

02_fetch_data.py

  • Purpose: Fetches non-coding regions of human genes directly from NCBI.
  • Configuration: You must edit the script to include your email for NCBI Entrez:
  • Usage: Can be triggered via the terminal to download specific genomic regions.

Phase 4: Prediction & Analysis

03_predict_dna.py

  • Purpose: Predicts potential motifs in a target FASTA file.
  • Input: You will be prompted for a path (e.g., DNA_file_holder\upstream_ACE2.fasta).
  • Output: Generates analysis_results\top_spam_candidates.fasta.

04_structural_check.py

  • Purpose: Performs structural validation on the candidates found by the prediction script.
  • Note: This reads analysis_results\top_spam_candidates.fasta. If you run a new prediction in step 03, this file is overwritten and refreshed for analysis.

🚀 How to Run

  1. Prepare your raw FASTA files in the positive_library and negative folders.
  2. Run the 00 scripts to build your training set.
  3. Run 01_ai_train.py to generate your model.
  4. Use 03_predict_dna.py followed by 04_structural_check.py to analyze new sequences.
